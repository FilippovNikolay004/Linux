# ОБЩЕЕ
pwd - Показать текущий путь
ls - Показать содержимое директории
[command] --help - Выводит подсказки и ключи определённой комманды
man [command] - Выводит мануал определённой комманды (q - чтобы выйти из мануала)
man -f [command or file] - Отображает номер раздела команды или файла
	Аналог man -f - это whatis
man -k [keyW] - Поиск команды по ключевому слову
	Аналог man -k - это apropos
info [command] - Выводит более детализированную информацию с возможностью навигации
	по разделам
whereis [command] - Выполняет поиск команд и страниц руководства
whatis [command] - Выводит краткую информацию по команде
grep [what] [where] - Поиск по ключевому слову в файле
wget [url] - Можно устанавливать файлы из интернета по URL
history - Отображает историю комманд
history [value] - Отображает определённое последнее кол-во комманд
![value] - Выполняет команду из списка по номеру
!! - Выполняет последнюю команду из истории команд
![command] - Выполняет последнюю заданную команду
!-[value] - Выполняет value команду снизу списка истории
whereis [nameA] - Отображает информацию где находятся файлы для определённого приложения
date - Выводит текущую дату
cal - Выводи текущую дату в виде календаря
cal [year] - Выводит календарь указанного года
echo '[value]' or [value] - Выводит текст на экран терминала
type [command] - Выводи информацию о команде (выводит расположение комады (не всегда, для решения, нужно исп. -а)). 
	Если отображает такое сообщение: [command] is a shell builton - то это встроенная команда в оболочку, 
	которая не требует доп. программ, а если отображает путь, то это внешняя программа (bin-файлы)
which [command] - Отображает полный путь к рассматриваемой команде (Просматривает через переменную PATH)
alias - Выводит псевдонимы, которые установлены в текущей оболочке
alias [nameMyCommand]=[command] or "[command]" - Создание своего алиаса (псевдонима) для запуска команды
	(После закрытия оболочки, новые псевдонимы теряются)
"" - Не позволяют интерпретировать некоторые метасимволы, но переменные в кавычках отображаться будут
'' - Не позволяют интерпретировать спец. символы, переменные и тд
`` - Используются для указания команды внутри команды. Пример: echo Today is `date` // Todat is Mon Nov 4...
\[metaCharacters] - Запрещает интерпретировать метасимволы 
[command1]; [command2]; [command3] - Позволяет поочерёдно запускать несколько команд
[command1] && [command2] - Позволяет запускать следующую команду, если первая выполнилась успешно
[command1] || [command2] - Если 1-я команда выполнилась успешно, то 2-я не выполняется
	Если 1-я команда не выполнилась успешно, то запускается 2-я команда "Либо запусти эту, либо вторую"

touch [nameF] or [path]/[nameF] - Создание файла
	touch [nameF] [nameF] - Создание нескольких файлов

	tail -n [value] [nameF] or [path]/[nameF] - Выводи определённое кол-во строк (благодаря ключу/флагу -n)
head [nameF] or [path]/[nameF] - Выводит первые 10 строк (по умолчанию)
	head -n [value] [nameF] or [path]/[nameF] - Выводи определённое кол-во строк (благодаря ключу/флагу -n)

mv [path]/[nameD or nameF] [path]/... - Перенос файлов/директорий (от куда, куда)
	mv [path]/[nameD or nameF] [path]/[nameD or nameF] - Переименовывает файл при переносе

# ПРИЛОЖЕНИЯ

sudo apt remove [nameA] - Удаление без удаления конфигурации программ
sudo apt purge [nameA] - Удаление с конфигурационными файлами
[nameA] - Запуск приложения
sudo apt update - Проверка на наличие обновлений
sudo apt upgrade - Выполнение обновлений
sudo apt-get check - Проверка целостности и зависимости (зависимости от библиотек и модулей)
sudo apt clean - Удаление загруженных архивных файлов
sudo apt autoclean - Удаление старых архивных файлов
sudo apt-cache search [nameA] - Выводи информацию о пакетах в репозиториях (поиск программ)
sudo list --installed - Выводит установленные пакеты (если используется apt)
